## 🚨 반드시 확인 (CLAUDE.md 과거 반복 버그)

### 1. 8분 제한 3중 검증 (CLAUDE.md §2)
- [ ] 검색창에 9분짜리 영상 키워드 → 결과에 '+신청' 버튼 클릭 → **"🎵 8분 이하 음악 컨텐츠만 재생 가능합니다" 토스트 뜨는지**
- [ ] URL 입력에 `https://www.youtube.com/watch?v=OcpgkIRQLb0` (9분짜리) 붙여넣기 → 같은 토스트 뜨는지
- [ ] 이미 추가된 곡이 실제로 9분으로 업데이트됐다면 재생 시점에 자동 제거되는지 (`_rtDurBlock`)
- [ ] 비음악 카테고리 영상(예: 뉴스, Vlog) 신청 시 차단되는지 (`checkVideoInfo` 카테고리 ID 10 체크)

### 2. 호스트-게스트 영상 동기화 (CLAUDE.md §1)
- [ ] 호스트 창 + 게스트 창 동시 열기 (다른 브라우저/시크릿 창 사용)
- [ ] 호스트가 **타임라인을 3:00 위치로 드래그** → 게스트 영상이 **3:00 근처에서 재생되는지** (처음부터 재시작 X)
- [ ] 호스트가 일시정지/재생 → 게스트도 동기화되는지
- [ ] 게스트 입장 순간 **음소거 아이콘 표시 + 실제로 소리 안 나는지** (iframe 로드 타이밍 버그 재발 여부)

### 3. 참여자 카운트 정확성 (CLAUDE.md §3)
- [ ] 호스트 혼자 방 만들면 **"1명"** 표시 (0명 아닌지)
- [ ] 게스트 1명 추가 입장 시 즉시 "2명"으로 갱신
- [ ] 게스트 탭 닫기 → 몇 초 내 "1명"으로 복귀 (`onDisconnect`)
- [ ] 호스트가 방 나가면 방 삭제되는지

### 4. Error 153 자동 스킵 (v1.30.0)
- [ ] 임베딩 금지된 영상이 큐에 있을 때 **"⚠️ 이 영상은 임베딩이 차단되어…" 토스트 + 다음 곡 자동 재생**
- [ ] 에러 영상 하나만 있을 때 `stopPlayer()` 호출되는지

## 🎯 핵심 기능 스모크

### 방 생성·입장
- [ ] 홈 → "플레이룸 만들기" → 닉네임 입력 → 방 생성 성공
- [ ] 6자리 코드 + QR 이미지 정상 생성 (`#qrcv canvas` 존재)
- [ ] "플레이룸 공유하기" 버튼 → 클립보드 복사 토스트
- [ ] 홈 → "코드 입력" → 6자리 입력 → 닉네임 → 입장 성공
- [ ] QR 스캔 시 자동 입장 (URL 파라미터 `?code=XXXXXX`)
- [ ] 비밀번호 있는 방: 틀린 PW → 거부 / 맞는 PW → 입장

### 신청곡
- [ ] 검색창에 키워드 입력 → 결과 리스트 표시 (썸네일+제목)
- [ ] '+신청' 클릭 → 큐에 추가 + "신청됨" 토스트
- [ ] URL 붙여넣기로 신청도 정상 동작
- [ ] 큐 비어있을 때 "아직 신청곡이 없어요 🎵" 표시
- [ ] 호스트가 곡 삭제 🗑 → 즉시 반영
- [ ] 호스트가 드래그로 순서 변경 → 모든 참여자 화면에 반영

### 재생 제어
- [ ] 자동 재생 / 반복재생 / 한곡만 모드 전환 (`cyclePlayMode`)
- [ ] 음소거 토글 🔊/🔇 → 아이콘 + 실제 소리 일치
- [ ] 볼륨 슬라이더 0~100 → 실제 볼륨 반영
- [ ] "영상+음악" ↔ "음악만" 모드 전환
- [ ] 재생 완료 시 다음 곡 자동 재생
- [ ] "재생 완료 곡 자동 삭제" 옵션 → 큐에서 즉시 제거

### 호스트 전용
- [ ] "신청곡 승인 필요" 켬 → 게스트 신청 → 승인 대기 카드에 쌓임
- [ ] 호스트가 승인 → 큐로 이동, 거절 → 사라짐
- [ ] 관리자 진입: Ctrl+Shift+A → PIN 9624 입력 → 플랜 스위처
- [ ] 모바일 로고 5-tap → 관리자 진입 (모바일 한정)
